Shared coding conventions allow teams to collaborate effectively. This rule checks that all file names match a provided regular expression.

As JSON files are used in a large variety of ways and by many frameworks, it might sometimes be tricky to enforce a strict file naming convention. However, the default regular expression has been designed to match as closely as possible the following guidelines:

  • Suffix files with .json
  • Only use letters, digits, '_' and '-' characters
  • Refrain from using '_' and '-' characters. Only use them when mandatory.
  • Extensively use Camel Case. For example, prefer: 
    myObject.json
    to 
    myobject.json
my_object.json
